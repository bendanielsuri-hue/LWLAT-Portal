{% comment %} Staff-source + search picker, shared by the meeting "Add Member" dialog and the Panel Group "Add Member" form. Params: school (optional, for the " Staff" source option + scoping), existing_staff_ids/existing_external_ids (ids already on the roster - sent to the server search as `exclude` so it can flag them "already a member" instead of hiding them), already_member_label (pill text, e.g. "Active Member" or "Already in Group"). The including page owns the Save/submit button and listens for the bubbling "member-picker:change" event to know when a selection is made. Results are server-fetched from the shared inclusion_panel_search endpoint (kind=staff|external) and hidden until typed - see InteractionLanguage.md "Search". The "New External Contact" trigger is NOT rendered here - it lives in the including page's own persistent footer (e.g. _panel_group_form_modal.html's .panel-group-modal-footer, alongside Back) so the two read as one aligned row instead of two stacked ones. The including page must provide a button carrying both `data-member-add-external` (this picker's own JS toggles its `hidden` attribute based on the External segmented option being active, scoped to the nearest ) and `data-member-add-external-toggle` (opens #external-contact-quick-add-dialog, wired in panel.js). {% endcomment %}
